# Epic 0 Evidence Register

## Purpose

This register is the index of record for Epic 0 baseline evidence and for inherited evidence generated by later epics.

Every evidence item must have:

- a stable ID
- date captured
- owner
- source artifact
- validation method
- review status

## Evidence Status Values

| Status | Meaning |
| --- | --- |
| Captured | Evidence exists and is linked |
| Reviewed | Evidence has been checked by the relevant reviewer |
| Superseded | Evidence is outdated and replaced by a newer item |
| Missing | Required evidence has not yet been captured |

| Category | Minimum requirement | Owner |

| Security review evidence | One reviewed item per changed auth/network/token surface | Epic owner |
| Privacy review evidence | One reviewed disclosure update per changed data flow | Epic owner |

## Review Requirements

1. Every reviewed evidence item must record the reviewer name or role.
2. Evidence without a reviewer may support active investigation but does not satisfy a gate.
3. Superseded evidence must point to the replacement item.
4. If evidence cannot be captured, the owning epic must record why and request a written exception.

## Escalation Rules

If a required evidence item is missing:

1. The work package cannot be marked complete.
2. The gate reviewer records the missing evidence in the review.
3. The epic owner either captures the evidence or records a debt acceptance request.
4. The Project Owner decides whether the gap is blocker, major, or minor.
